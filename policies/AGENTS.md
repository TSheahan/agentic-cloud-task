# policies/

Reusable methodologies and conventions that govern how work is done in this
project. Load on demand when a task intersects the policy's scope.

| Policy | Scope | Read when |
|--------|-------|-----------|
| [state-convergence-pattern.md](state-convergence-pattern.md) | Three-section (Target State / Apply / Audit) mixed agentic-imperative-declarative convergence methodology | Writing or reviewing any profile that converges a system to a specified state |
| [brain-dump.md](brain-dump.md) | Dump-accumulate-recompose sequence for transferring substantial user domain knowledge | User has a large body of knowledge to transfer and structured elicitation would fragment it |
| [agent-user-co-ownership.md](agent-user-co-ownership.md) | Bilateral agent–user contract for structure, rules, transparency, and session boundaries | Changing repo layout, routing, policies, or when grounding agent behavior in co-ownership |
| [continual-learning.md](continual-learning.md) | Light-touch bias toward structurally legible replies (not extra narration) | Any task where agent explanations should stay aligned with project structure and routing |
